## 📊 주요 발견사항

### **1. 극도로 문제가 되는 변수들**
- **발생액**: 스케일이 10^10 단위로 극도로 크고, 분포가 매우 비대칭
- **매출액_대비_현금흐름**: 높은 음의 왜도로 극값이 존재

### **2. YoY 성장률 변수들의 특성**
- 모든 성장률 변수들이 높은 변동성과 첨도를 보임
- 경기 변동과 기업별 특성으로 인한 극값들이 다수 존재
- RobustScaler 적용 필수

### **3. 변화량 변수들의 특성**
- 부채비율_변화량, 유동비율_변화량 등 모든 변화량 변수들이 높은 변동성
- 기업의 급격한 구조 변화나 특수 상황으로 인한 극값 존재

### **4. 기본 재무비율의 안정성**
- Altman Z-Score 기반 기본 변수들은 상대적으로 안정적
- 표준화된 회계 기준으로 인해 분포가 비교적 정규분포에 가까움

## 💡 실무 적용 권장사항

### **1. 모델링 전 전처리**
```python
from sklearn.preprocessing import RobustScaler, StandardScaler

# 고위험 변수들 (RobustScaler)
high_risk_vars = ['발생액', '매출액_대비_현금흐름', '운전자본회전률_변화량', ...]
robust_scaler = RobustScaler()

# 안정적 변수들 (StandardScaler)  
stable_vars = ['TLTA', 'WC_TA', 'SIGMA', ...]
standard_scaler = StandardScaler()
```

### **2. 이상치 처리 고려사항**
- **발생액**: 극값 제거 또는 로그 변환 후 스케일링 고려
- **성장률 변수들**: Winsorization (상하위 1-5% 절단) 적용 고려
- **변화량 변수들**: 절댓값 기준 상위 5% 극값 검토 필요

### **3. 모델별 스케일링 전략**
- **선형 모델**: StandardScaler 우선, 이상치 많은 변수만 RobustScaler
- **트리 기반 모델**: 스케일링 불필요하지만, 해석 목적으로 적용 고려
- **신경망**: 모든 변수에 스케일링 필수, RobustScaler 권장
